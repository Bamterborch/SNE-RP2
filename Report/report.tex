\begin{abstract}
Generating packets in order to fill the links capacity is not hard to accomplish. 
Especially for UDP traffic since it doesn't do session synchronization nor congestion control and does not offer retransmission of data. 
For a session oriented reliable protocol such as TCP - which ensures that the data sent is actually received by the target, it is much more complex to achieve. 
For session-based traffic the techniques implemented in a standards-compliant TCP stack make it harder to reach the limits of a device in terms of raw bandwidth.
Due to the techniques used by TCP, which do not allow the capacity of a link to be overloaded, the amount of packets per second and the number of concurrent sessions are possible limitations this research is focusing on in order to get the limitations of hardware and software.  
Before new systems or services go into production, an engineer needs to know the limitations of the hardware or software to make sure alerts are raised at the correct thresholds.
This research investigates the suitability of various load testing tools - with and without the use of the Data Plane Development Kit - and apply these tools to various end-to-end network topologies and hardware systems between a client and a target. 
The report demonstrates that the weakest links in the path can be identified and its load tolerance limits can be found by using the tools examined during this research.
\end{abstract}
